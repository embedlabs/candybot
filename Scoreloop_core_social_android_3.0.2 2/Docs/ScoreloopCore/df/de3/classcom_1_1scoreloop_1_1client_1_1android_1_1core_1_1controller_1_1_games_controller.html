Detailed Description

The GamesController requests lists of Game objects from the server. Five different lists may be retrieved:

  • list of Scoreloop-enabled games played by the user's buddies
  • list of Featured Scoreloop-enabled games
  • list of the most popular Scoreloop-enabled games
  • list of new Scoreloop-enabled games
  • list of games that have been played by a specific user.

Basic Usage:

To retrieve a list of games from the server:

  1. Get an instance of GamesController.
  2. Set the range by calling the setRangeLength() method. This is the range of games that will be requested from the server.
  3. Make the server request using either:
    • loadRangeForBuddies
    • loadRangeForFeatured
    • loadRangeForPopular
    • loadRangeForNew
    • loadRangeForUser
  4. Await notification of success via delegate callbacks.
  5. After a successful request, access the retrieved games by calling the getGames() method which would return list of games.

To page through screens of games:

  1. Use the controller to check if it is possible to paginate by checking either:
    • hasNextRange
    • hasPreviousRange
  2. If pagination is possible, call either:
    • loadNextRange
    • loadPreviousRange as required.
  3. Await notification of success via delegate callbacks.
  4. After a successful request, access the retrieved games by calling the getGames() method which would return list of games.

To return to the first range of retrieved games: call loadFirstRange.

NOTE: Make sure that the methods: loadNextRange, loadPreviousRange and loadFirstRange are called only after one of the other load methods:

  • loadRangeForBuddies,
  • loadRangeForFeatured,
  • loadRangeForPopular,
  • loadRangeForNew,
  • loadRangeForUser,

is called.

Member Function Documentation

List<Game> getGames ( )

This method returns the List of Game objects that were retrieved from the server by calling one of the controllers load methods. This method will return an empty list if no request to the server has been made, or if the server request failed.

Returns:
java.util.List<Game> The list of game objects retrieved from the server.
boolean getLoadsDevicesPlatformOnly ( )

Returns whether loading games will restrict them to the platform (e.g. iPhone, Android, etc. ) of this device. Default is true.

Returns:
true, if only games of the devices platform are loaded, false, if games for all platforms will be returned.
int getRangeLength ( )

This method returns the length of the currently loaded range. It returns a length of 0 if no range has been loaded.

Returns:
int, length of the currently loaded range.
boolean hasNextRange ( )

This method returns if a next range is available or not. It returns YES if the controller is ready and if there is a next range available, otherwise NO. This is used to check if forward pagination is possible.

Returns:
boolean, specifying if a next range is available or not.
boolean hasPreviousRange ( )

This method returns if a previous range is available or not. It returns YES if the controller is ready and if there is a previous range available, otherwise NO. This is used to check if backward pagination is possible.

Returns:
boolean, specifying if previous range is available or not.
void loadFirstRange ( )

This load method retrieves the first set of Scoreloop-enabled games from the Scoreloop server. The set of games retrieved from the server will depend on the previous load method called. For example, if the load method that was called earlier was loadRangeForNew, then loadFirstRange would return the first set of new Scoreloop-enabled games.

Note: Make sure that this method is called only after one of the other load methods: loadRangeForBuddies, loadRangeForFeatured, loadRangeForPopular, loadRangeForNew, or loadRangeForUser is called.

void loadNextRange ( )

This method requests the next available range of Scoreloop-enabled games from the server.

This method is used to "page forward" through the list of games.

Note: Make sure that this method is called only after one of the other load methods: loadRangeForBuddies, loadRangeForFeatured, loadRangeForPopular, loadRangeForNew, or loadRangeForUser is called.

void loadPreviousRange ( )

This method requests the previous range of Scoreloop-enabled games from the server.

This method is used to "page backwards" through the list of games.

Note: Make sure that this method is called only after one of the other load methods: loadRangeForBuddies, loadRangeForFeatured, loadRangeForPopular, loadRangeForNew, or loadRangeForUser is called.

void loadRangeForBuddies ( )

This load method retrieves the list of Scoreloop-enabled games played by the user's buddies.

Based on two indicators hasPreviousRange and hasNextRange it is possible to navigate through the whole list.

void loadRangeForFeatured ( )

This load method retrieves the list of Featured Scoreloop games from the server.

Based on two indicators hasPreviousRange and hasNextRange it is possible to navigate through the whole list.

void loadRangeForNew ( )

This load method retrieves the list of new Scoreloop-enabled games from the server. All the recently created scoreloop games will be returned.

Based on two indicators hasPreviousRange and hasNextRange it is possible to navigate through the whole list.

void loadRangeForPopular ( )

This load method retrieves the list of the most popular Scoreloop-enabled games from the server.

Based on two indicators hasPreviousRange and hasNextRange it is possible to navigate through the whole list.

void loadRangeForUser ( final User  user)

This method requests the range of Scoreloop-enabled games from the server that have been played by the given user.

Based on two indicators hasPreviousRange and hasNextRange it is possible to navigate through the whole list.

NOTE: This method (as opposed to the other load methods like loadRangeForNew()) does not respect the getLoadsDevicesPlatformOnly() flag and always loads games across all platforms.

Parameters:
userThe user for whom the request will be made.
