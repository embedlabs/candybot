Detailed Description

A class that holds information about a payment of a user.

A Payment instance acts as a receipt for the transaction that was carried out on the Scoreloop server. As the class only records the transaction that was carried out on the server, it is not a receipt for the real-world payment.

The payment of a user could be in one of the following states: BOOKED, CANCELED, CREATED, FAILED, PENDING, or REFUNDED.
