Users

Q. What is meant by a user?
A user refers to a player of your game application.

Q. What happens the first time I connect to Scoreloop?
The first time a user connects to Scoreloop they are assigned a unique immutable identifier that is used in all subsequent connections.
New users also receive an automatically generated username, (which has the form player_XXXX). Users are encouraged to change this to something more personal, (see the User Integration Guide for details about how to implement a user-profile update function).

Q. How do I login to Scoreloop?
Logging into Scoreloop is an automatic process that happens whenever a user playing a Scoreloop-enabled game makes contact with our servers.
There is no explicit step where users are required to enter their username or password.

Q. What information is contained in my user profile?
Your user profile contains, (at a minimum), your Scoreloop username. This will either be the username automatically generated for you by Scoreloop the first time you connected, or else a username you have chosen for yourself. Additional, optional information that can be contained in the profile include: your email address; your date of birth; a list of your Scoreloop friends, among other things. You shoudl read the User API documentation for more information about the type of information that can be stored there.

Q. Can I change the information in my profile?
Yes. You are free to change or update your username, email address, and date of birth. You are also free to add or remove other users from the list of your Scoreloop friends. See the User Integration Guide for more details about how to implement functions to update the user profile. Other profile properties are updated automatically by the Scoreloop server, (whether the user is able to partake in a Scoreloop challenge, for example). You should see the User API documentation for more details.

Q. What is meant by the term 'session user'?
This is the user associated with the device that is currently connected to Scoreloop. See the API documentation for the Session for more information.

Q. What classes are involved in user management?
Scoreloop users are modelled using the User class. User objects are managed using two distinct data controllers:

  • The UserController manages single instances of the User class and is used for:
    • setting user details,
    • retrieving a single user's details from the server,
    • updating a single user's details on the server.
  • The UsersController manages lists of User objects and is used for:
    • searching for other users on the server,
    • filtering search results according to some criteria.

Q. What other classes are users involved in?
Users are also associated with the following classes and APIs. You should note that the values of User properties can be dependent on where the object comes from, (for example, some user properties will only have meaningful values if the user is part of a Scoreloop challenge). You should read the accompanying API documentation for more information:

  • The Score class:
    • Score.getUser()
  • The Challenge class:
    • Challenge.getContender()
    • Challenge.getContestant()
    • Challenge.getWinner()
    • Challenge.getLoser()
  • The Session class:
    • Session.getUser()

Q. What are 'buddy users'?
A user may designate other Scoreloop users as friends and add them to a 'buddy-list'. A user's buddy-list can be used to generate a leaderboard composed of the best scores submitted by their friends. You should see the User Integration Guide to see how to implement a function for adding other users to a buddy-list.

Q. Can multiple users share the same device ?
No. Multiple Users cannot use the same device to connect to Scoreloop. Every device that is used to play a Scoreloop enabled game is uniquely associated with a single specific user.

Q. Can multiple devices share the same User Profile?
Yes, multiple devices can share the same Scoreloop User Profile. This means that a user is free to use the same Scoreloop profile to play games on their smartphone and on their tablet computer, for example.

Q. Can I associate some custom data with a Scoreloop User ?
Yes, you can associate additional custom data with the User using the User Context. There are methods like setContext() and getContext() available to access the User context. Use the UserController's submitUserContext() and loadUserContext() methods to submit and load the context to and from the Scoreloop Server.

Q. Is there a limit on how much data can be stored in the User context?
We don't impose any hard limits on how much data can be stored in the user contexts.

Q. Does the User context get downloaded whenever a User object is downloaded or is it just retrieved when User.getContext() is called ?
The User context associated is only retrieved from the server when you explicitly call UserController.loadUserContext() . Similarly, the context gets submitted to the server only when you explicitly call UserController.submitUserContext() .
