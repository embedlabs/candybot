Detailed Description

A RSSFeed allows you to retrieve feed items from the Scoreloop server to be displayed in your application.

Feed items could be either:

  1. news items explicitly specified by you on the Scoreloop admin site - e.g. to push promotional information to the game users, or
  2. activities of other users of the game automatically generated by the Scoreloop server, or
  3. both of the above.

Developers are free to configure on https://developer.scoreloop.com which of the above options they would prefer.

Retrieval of feed items is modeled in an asynchronous way. When constructing a RSSFeed you have to pass a RSSFeedObserver in order to receive callbacks when the next RSSItem is ready.

To retrieve the next RSSItem: call the main API of this class called requestNextItem(). This loads the latest feed from the server - if not done before - and then returns the next unread RSSItem of this feed via the observer callback methods. A policy parameter to this method determines which item should be returned next from the feed.

In order to load feeds afresh from the server, you will have to place a call to reloadOnNextRequest() before the next call to requestNextItem().

Basic Usage:

  1. Create an RSSFeed Object by calling RSSFeed(). You will need to pass a RSSFeedObserver as an argument to the constructor in order to receive callbacks.
  2. Call the method requestNextItem() to load the latest feed from the server. You will have to pass a policy which suits your criteria on how you would like to filter out the RSSFeed Items. You could use the pre-defined policies STANDARD_POLICY , STICKY_POLICY , UNREAD_POLICY or ON_START_POLICY . You also have the option of defining your own policy. See Policy and ChainedPolicy for more details.
  3. Await server response via observer callback methods. See RSSFeedObserver for more details.
  4. Display the received feedItem in the UI.
  5. Go to Step 2 and repeat 2,3,4 till there are no more Items returned.
  6. Call reloadOnNextRequest() and then go to Step 2.

void cancelRequestNextItem ( )

This method cancels any call that has been placed to requestNextItem(). If the request has already completed, this method will do nothing. If the request is ongoing, then this method will call the RSSFeedObserver.feedDidFailToReceiveNextItem() callback with an RequestNextItemCanceledException .

void reloadOnNextRequest ( )

Calling this method indicates that the newsfeed should be reloaded before the next RSSItem is requested. Call this method when there are no further items returned by calling requestNextItem(). This method:

  • Resets the current feed to null.
  • Clears the set of all "read" items.
  • When requestNextItem() is next called the feed will be reloaded.

void requestNextItem ( final Policy  policy)

Requests the next RSSItem. Calls RSSObserver's feedDidRequestNextItem() synchronously and either of RSSObserver's feedDidReceiveNextItem() or feedDidFailToReceiveNextItem() after attempting to retrieve the next news-item.

The news-feed is only read at first call to this method or every time after you invoked reloadOnNextRequest(). In these cases the finishing callback methods get called asynchronously but in the same thread as the thread you place this call. In the other cases the news-feed is cached and the finishing callbacks are called synchronously before this method returns.

Parameters:
policya required Policy object which describes the filtering policy for news-items. There are some pre-defined ones STANDARD_POLICY ,UNREAD_POLICY , STICKY_POLICY or ON_START_POLICY . You could check those out or you could also implement your own policy and pass it to this method.
void setDefaultItem ( final RSSItem  item)

Use this method if you want requestNextItem() to always report a non-null RSSItem object when used together with the STANDARD_POLICY .

Parameters:

Member Data Documentation

final Policy ON_START_POLICY = new ChainedPolicy(new Policy[] { UNREAD_POLICY, STICKY_POLICY }) [static]

A Policy defined to filter those items which are sticky and have not been read yet. It is a chained policy of UNREAD_POLICY and STICKY_POLICY .

final Policy STANDARD_POLICY = new StandardPolicy() [static]

A policy defined to filter news items in the following order:

  1. First, the sticky items which have not been shown during this session
  2. Then, the non-sticky items which have not been read. Uses the isUnread() method to check if the item has been read or not
  3. and then finally, the item returned from getDefaultItem() if it is present. You could use the method setDefaultItem() to set the default item that needs to be returned
final Policy STICKY_POLICY = new StickyPolicy() [static]

A Policy defined to filter those RSSItems which have been marked as Sticky. You could use isSticky() method of the RSSItem class to check if the RSSItem as been marked as sticky. Developers could mark the items as sticky on https://developer.scoreloop.com.

final Policy UNREAD_POLICY = new UnreadPolicy() [static]

A policy defined to filter those items that are unread. Items for which isUnread() is true.
