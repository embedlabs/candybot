Initialization

The first step is to do the initialization. An instance of the Client needs to be created which would set up the underlying infrastructure necessary for using Scoreloop in a game. When a Client is instantiated, a session instance will be created transparently.

The following table below gives an overview of the classes (including the purpose) associated with Initialization.

ClassPurpose
Client
  • Sets up the infrastructure to use Scoreloop

Steps:

Before performing these Steps please make sure you have followed the instructions on Project Set-up. and saved the scoreloop.properties file in the assets directory of your project. You should also have your Game's secret noted from the developer website.

The scoreloop.properties file you downloaded will include your Game's name, version and ID.

If your game supports modes, you will need to add both the minimum and maximum modes to the scoreloop.properties file. The following example shows how to configure the scoreloop.properties file for a game that has 3 modes: 

 game.mode.min = 0
 game.mode.max = 3


Note that the maximum mode property is not inclusive, (that is, the above example defines modes 0, 1 and 2).

Create the Scoreloop Client

A good place to initialize the Client is within your game's application class. This is the class that will normally extend the android.app.Application. Within your game's application class the recommended place to initialize the Scoreloop Client is within the onCreate() method, to which you only have to pass the android.content.Context.

Add the following line to your Application's onCreate method: 

                // initialize the client using the context and game secret
                Client.init(this, secret, null);
