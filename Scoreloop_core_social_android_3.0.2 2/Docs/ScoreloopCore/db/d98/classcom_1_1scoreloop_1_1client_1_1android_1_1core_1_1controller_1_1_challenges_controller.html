Detailed Description

The ChallengesController retrieves lists of challenges on behalf of the current session user from the Scoreloop servers. Two separate lists may be retrieved:

  • the list of challenges currently available to the user to play
  • the user's challenge history (the list of all challenges associated with the user, e.g. completed, current, rejected, etc)

Basic Usage:

  1. Set the mode for the controller, if required.
  2. Use loadOpenChallenges() or loadChallengeHistory() to request the appropriate challenge list.
  3. Await a successful server response via delegate callbacks.
  4. After a successful server request, use getChallenges() to access the retrieved challenges.

Member Function Documentation

List<Challenge> getChallenges ( )

Returns the list of all challenges that have been requested from the server. The list will contain either the list of all open challenges available to the user to play, or the user's challenge history, depending on the type of request made by the controller.

If this method is called before a request to the server has been made, an empty list will be returned.

Returns:
A java.util.List of Challenge objects.

void loadChallengeHistory ( )

Requests all challenges associated with the user. These include:

  • all completed challenges,
  • all direct challenges for the user,
  • any direct challenges created by the user but that were rejected.

When this method is called the RequestControllerObserver will be called asynchronously.

Use getChallenges() to access challenges after a successful request.

void loadOpenChallenges ( )

This method requests all challenges which may be played by the current session user. If any direct challenges exist for the user, these will be requested first. If no direct challenges exist, then all open challenges will be requested instead.

When this method is called the RequestControllerObserver will be called asynchronously.

Use getChallenges() to access challenges after a successful request.
