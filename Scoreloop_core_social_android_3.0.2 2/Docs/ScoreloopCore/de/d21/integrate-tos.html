Terms of Service

Whenever a player is about to establish a connection with Scoreloop for the first time, a Terms Of Service dialog will pop up. The user has the accept it so as to continue to use Scoreloop. Please note that the Terms Of Service is per Game. If the User has accepted the Terms once for a particular game, the dialog will not be shown again for that game. However, he will have to accept it again in another game. However, if the user has rejected the Terms, he will not be allowed to continue interacting with Scoreloop. Game developers have to ensure that the players have accepted the Terms.

The following table below gives an overview of the classes (including the purpose) associated with "Terms Of Service".

ClassPurpose
Terms Of Service
  • Model class for Scoreloop Terms of Service
  • Provides status on the Terms Of Service
TermsOfServiceController
  • Manage the terms of Service
Session
  • Access to Terms Of Service related information

Terms Of Service related information can be accessed through the Session as follows: 

    TermsOfService tos = Session.getCurrentSession().getUsersTermsOfService();
    // tos.getStatus() is one of Status.ACCEPTED, Status.PENDING, Status.REJECTED, Status.OUTDATED

Basic Usage:

To invoke the TOS dialog if it was not accepted previously, the following code may be used: 

 final TermsOfServiceController controller = new TermsOfServiceController(new TermsOfServiceControllerObserver() {
 @Override
 public void termsOfServiceControllerDidFinish(final TermsOfServiceController controller, final Boolean accepted) {
    if(accepted != null) {
        // we have conclusive result.
        if(accepted) {
            // user did accept
        }
        else {
            // user did reject
        }
    }
 }
 });
 controller.query(activity);
