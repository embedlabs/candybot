Detailed Description

RequestController is the base class for all controllers that communicate with Scoreloop.

Member Function Documentation

Exception getError ( )

Returns an exception object if the last request failed. When a new request is issued, this exception object will be nullified at the same time.

Returns:
the same exception object as reported in a call to RequestControllerObserver#requestControllerDidFail(RequestController, Exception) on the last request, or null, if the last request succeeded.
boolean isCachedResponseUsed ( )

Reports whether the response caching is used or not on the next requests. See also setCachedResponseUsed()for more information.

Returns:
true, if the response caching is used, false if the servers would be touched on the next requests.
void setCachedResponseUsed ( final boolean  useCache)

Use this setter to enable or disable the use of cached responses for the next requests.

When set to true (which is the default), the SDK caches the responses of requests made before and reuses them when subsequent requests with the same parameters are made within some internally defined time intervals. This reduces the need to communicate with the Scoreloop servers in many cases.

The response cache will be flushed automatically once any submit call is triggered (like UserController#submitUser() ).

When you want to enforce to make a server call instead of using a cached response value on the next asynchronous calls, pass false to this method.

Parameters:
useCachetrue, if you want to use cached response values, false if you want to reach the servers.
