Detailed Description

The Score class models the score achieved by a particular user in a particular game session. Score objects must minimally contain a result, which is typically the main numerical outcome achieved by users of the game.

Score objects may optionally include:

  • a minor or secondary result,
  • a level (indicating the game stage at which the Score was achieved),
  • a mode (indicating the gameplay setting at which the Score was achieved).

The result, minor result, and level are used for score comparison purposes. The comparison schema can be configured by the developer at https://developer.scoreloop.com..

Modes are important as Scoreloop generates separate leaderboards for each mode defined for a game.

Score objects also have a unique identifier generated and assigned by Scoreloop. This is assigned to the object when the Score is submitted to the Scoreloop servers.

Scores are managed by two data controllers: the ScoreController which manages single instances of Score; and the ScoresController which manages lists of Score objects.

Score objects may also contain information about score rank, which is the position of the score on a Scoreloop leaderboard. However, the rank of a score will generally only have a meaningful value when it is retrieved from the server using a ScoresController

Basic Usage:

Score objects are returned in the following scenarios:

  1. After calling the constructor below, (Score(final Double, final Map<String, Object>)).
  2. From a ScoreController, (by calling getScore()).
  3. From a ScoresController. After the controller has requested scores from the server, the list of Score objects can be accessed by using the calling getScores().

A Score needs to have a result [setResult(), getResult()] set to be submitted. The result is the primary value for the score and will be used to order scores on a leaderboard.

The following properties can optionally be set before submitting the score:

  • mode [setMode(), getMode()] is used to differentiate between types of gameplay, while keeping seperate leaderboards for each mode.
  • level [setLevel(), getLevel()] can be used to indicate a level. Score values should carry over between levels, since all levels will appear on the same leaderboard.
  • minorResult [setMinorResult(), getMinorResult()] is a secondary result parameter, that can be used to provide aditional information about the score and can be used as a comparison parameter for two scores with the same primary result.
  • context [getContext(), setContext(Map)] can store arbitrary values associated with the Score.

The following properties will be set after the Score has been submitted to the Server:

  • identifier [getIdentifier()]
  • rank [getRank()]
