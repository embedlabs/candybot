Detailed Description

The UsersController requests lists of User objects from the server, based on a variety of different criteria:

  • user login,
  • user email,
  • the local address book,
  • a social provider.

Basic Usage:

  1. Set the match limit for the controller using setSearchLimit(). The search request will stop returning User objects once this limit has been reached and will only return the number of matches instead. This number is also limited to 999.
  2. Set the flag indicating whether the search should extend beyond users of the current game, setSearchesGlobal().
  3. Make the server request by calling one of:
    • UsersController.searchByEmail(String),
    • UsersController.searchByLocalAddressBook(),
    • UsersController.searchByLogin(),
    • UsersController.searchBySocialProvider(),
    • UsersController.loadRecommendedBuddies(),
    • UsersController.loadBuddies().
  4. Await a successful server response via delegate callbacks.
  5. Call UsersController.getUsers() to access the list of users retrieved from the server.
  6. If required, access the total count of positive matches on the server using UsersController.getCountOfUsers().

If the number of users matched on the server exceeds the match limit set by the developer using UsersController.setSearchLimit(), the following happens:

  • isOverLimit() will return true.
  • Matched users will be the empty list.
  • Only the number of matched users will be accessible via UsersController.getCountOfUsers().
  • The count of matched users is restricted to a maximum of 999. You can whether this has been reached by calling isMaxUserCount().

int getSearchLimit ( )

Accessor that returns the maximum number of User objects to be matched per server request. The search request will stop once this limit has been reached.

If the number of positive matches on the server exceeds this limit then the matched user objects will not be accessible. This means that calling getUsers() in these cases will return an empty list. However, the count of positive matches will still be accessible via getCountOfUsers(). This count is also restricted to isMaxUserCount().

Currently defaults to 25.

Returns:
java.lang.Integer object corresponding to this limit.

void searchByLocalAddressBook ( )

This method gathers email addresses from the local address book and sends them to Scoreloop. Addresses are sent to the server in a hashed form to safeguard user privacy.

The search operation then attempts to match the local hashed email addresses with the email addresses of existing Scoreloop users stored on the server.

After a successful server request, the retrieved User objects can be accessed by calling getUsers(). The total count of positive matches found on the server can be accessed by calling getCountOfUsers().
