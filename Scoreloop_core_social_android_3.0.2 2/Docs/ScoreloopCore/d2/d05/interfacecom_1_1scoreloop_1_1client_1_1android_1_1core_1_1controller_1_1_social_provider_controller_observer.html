Detailed Description

The SocialProviderControllerObserver class handles delegate callbacks when connecting to social networks. Callbacks will be triggered in the following circumstances:

  • connecting to the social provider is cancelled,
  • the user entered the wrong username or password for the social network,
  • the connection failed,
  • the connection succeeded.

This method is called when it is clear that the authorization error is due to the failure of the user to provide valid credentials.

If a custom UI is present, then it will be dismissed. No other callback will be issued.
