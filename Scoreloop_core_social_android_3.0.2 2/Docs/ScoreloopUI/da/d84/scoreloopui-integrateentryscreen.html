Showing the Scoreloop Entry Screen

It is recommended that you include a link to the Basic Scoreloop Entry Screen from the main menu of your game as it provides a central
screen from which users can navigate to other screens. The following code shows an example of how you might go about doing this.

 //You should replace button_entry with the appropriate id name from your project
 final Button entryButton = (Button) findViewById(R.id.button_entry);

 entryButton.setOnClickListener(new View.OnClickListener() {
     
        @Override
        public void onClick(final View v) {
 
             startActivity(new Intent(this, EntryScreenActivity.class));
     
        }
  });

Scoreloop Buddies and The Address Book

By default, ScoreloopUI enables access to the local device address book. This lets ScoreloopUI find other Scoreloop users from the address
book and add them to your buddy-list. If you wish to disable this feature, you will need to modify the scoreloop.properties file as shown:

 ui.feature.address_book = false

If you disable this feature you should also remove the READ_CONTACTS permission from the Android Manifest (unless your game needs
it for other purposes). The following line may be removed from the Android Manifest: 

 <uses-permission android:name="android.permission.READ_CONTACTS" />

See the Updating the Android Manifest section for more details. Continue with Score Submission.
