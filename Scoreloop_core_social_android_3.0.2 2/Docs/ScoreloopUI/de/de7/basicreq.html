Essentials

Basic Requirements:

  • Android 1.6
  • Java 1.5

Assumptions:

  • You have already downloaded and installed an appropriate IDE.
    (Eclipse is recommended. Go to http://www.eclipse.org for details).
  • You have already downloaded and installed the Android SDK and configured your IDE for Android development.
    (Go to http://developer.android.com for more details).

Test Conditions

ScoreloopUI has been developed in and tested against the Android SDK 10 and ADT v. 10.0.1. It is strongly recommended that you use this in development as it is likely you will encounter problems if you user lower versions of the SDK. If you do experience issues in development or compilation, please get in touch with us via our support forum http://support.developer.com .

Notes on Integration

A basic integration of ScoreloopUI provides access to:

  • a customizable user profile,
  • high-scores and leaderboards,
  • a personal "buddy list" for each user,
  • access to other Scoreloop-enabled games via the Scoreloop Social Market.

These features are available via the Scoreloop Entry Screen.

Initialization of ScoreloopUI should take place in the Android application class, (as opposed to from the MainActivity, for example). This ensures that initialization of the ScoreloopManager takes place only once. Also take care to destroy the ScoreloopUI from the Android application class so as to ensure proper termination of the ScoreloopUI. The section on Initialization and Destruction provides practical examples of how to do this.

We recommend that you include a link to the ScoreloopUI Entry Screen from your game's main menu. This screen provides an excellent entry point to a range of Scoreloop features and includes links to the following game related features:

  • the Leaderboards view,
  • the User Profile view, and
  • the Buddies view.

It is recommended that any ScoreloopUI APIs are called exclusively on the main thread (a.k.a. UI-Thread). ScoreloopUI APIs are designed to be asynchronous and use an internal worker thread for their communication with the Scoreloop server. This means that the main thread will not be blocked even if an API call is in progress. When a request is ready Scoreloop ensures that the callback methods are also called on the main thread, which means that you can immediately update the UI or invoke additional ScoreloopUI calls. If your game uses additional threads, do not call any ScoreloopUI APIs directly from them. In these cases it is better to use Android Handler objects, (see android.os.Handler to call the ScoreloopUI API from the main thread, (as you would when performing common UI tasks, for example).
