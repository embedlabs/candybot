Supporting Multiple Modes

Modes refer to different gameplay settings such as speed or difficulty. By default, Scoreloop assumes that every game consists of at least one mode. To support multiple modes in your game,(for "slow, medium, fast", perhaps), you will need to do the following:

1. Configure the scoreloop.properties file so that it correctly reflects the number of modes you have defined in your game. In the properties file you need to define the minimum mode in your game, as well as the maximum mode. You should note that the maximum mode is a non-inclusive value. To define 3 modes in your game, (mode 0, mode 1, mode 2), you should add the following to the properties file:

game.mode.min = 0
game.mode.max = 3

2. If your game has more than 1 mode, you need to name the different modes as well otherwise it might result in a exception. To name the modes and display it correctly, you will need to modify the scoreformatter option in your scoreloop.properties file. The example snippet below defines 3 modes: Mode A, Mode B, and Mode C. You should of course choose mode names that are meaningful for your game. 

 ui.format.score = mode:0=A;mode:1=B;mode:2=C;

As an example, if you want to configure the modes as Esay, Medium and Hard, the scoreloop.properties file should be defined as follows: 

 ui.format.score = ModeSymbol=;mode:0=Easy;mode:1=Medium;mode:2=Hard;

By default the ModeSymbol is "Mode", however when you specify "ModeSymbol=;" as shown above, it overwrites the default mode Symbol (in this case with an empty string, so the modes would be displayed as "Easy", "Medium", "Hard" instead of "Mode Easy", "Mode Medium", "Mode Hard").

For more details refer the ScoreFormatter in the basic integration section.

NOTE: Developers who have used modes.xml may continue to use it, instead of modifying "format.score" in the scoreloop.properties file. However, please note that using of modes.xml has been deprecated and will not be supported in future releases.

3. To submit the score for the specific mode, you need to call the ScoreloopManager.onGamePlayEnded(Double, Integer) method with the appropriate mode. 

 //This example assumes the existence of scoreResult and mode, the value of which are obtained during gameplay.
 ScoreloopManagerSingleton.get().onGamePlayEnded(scoreResult, new Integer(mode));

4. When launching the leaderboards view ScoreloopUI displays the leaderboard in mode zero by default. To change this default behaviour so that a different mode is used, you will need to set the MODE property of the LeaderboardsScreenActivity when creating the android.content.Intent:

final Intent intent = new Intent(this, LeaderboardsScreenActivity.class);

//specify the mode you want the leaderboard to be opened in
intent.putExtra(LeaderboardsScreenActivity.MODE, 1); 

startActivity(intent);

See the Custom Leaderboards Integration Guide for more detailed information about where the best place to do this is.
