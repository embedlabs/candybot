Detailed Description

The OnScoreSubmitObserver receives notifications and callbacks from the server after a score has been submitted to Scoreloop.

The observer must be correctly set in the ScoreloopManager class before a score submission is made.

Basic Usage:

  1. Implement the onScoreSubmit(final int, final Exception) method.
  2. Set the observer using ScoreloopManager.setOnScoreSubmitObserver(OnScoreSubmitObserver).

Member Function Documentation

void onScoreSubmit ( final int  status,
final Exception  error 
)

This method is called after a score has been submitted to the server.

It will normally be implemented within the gameplay. In the case of a successful response:

  • the developer should display a toast notification if required; ScoreloopUI does not display a message by default.
  • the developer may wish to open an activity to post the score to Facebook or Twitter, see Scores and Leaderboards for more details..

In the case of an unsuccessful response ScoreloopUI will display an error notification.

Parameters:
statusThe status code, can be either STATUS_SUCCESS_SCORE , STATUS_SUCCESS_CHALLENGE , STATUS_SUCCESS_LOCAL_SCORE , STATUS_ERROR_NETWORK , or STATUS_ERROR_BALANCE .
errorjava.lang.Exception If the submission was unsuccessful this object will hold the reason for the error.
