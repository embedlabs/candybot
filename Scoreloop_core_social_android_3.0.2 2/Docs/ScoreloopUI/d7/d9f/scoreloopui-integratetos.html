Terms Of Service

Whenever a player is about to establish a connection with Scoreloop for the first time, a Terms Of Service dialog will pop up.

The user has the accept it so as to continue to use Scoreloop. If the User has accepted the Terms once, the dialog will not be shown again. It is shown only the first time. However, if the user has rejected the Terms, he will not be allowed to continue to interacting with Scoreloop. Game developers have to ensure that the players have accepted the Terms.

See the following examples on how to control when to query the user and how to determine if the user previously rejected the Terms.

At any place, where you want to make sure the user has accepted the TOS call, you would do something like this: 

 ScoreloopManagerSingleton.get().askUserToAcceptTermsOfService( [your current Activty here], new Continuation<Boolean>() {
    public void withValue(final Boolean value, final Exception error) {
        if (value != null && value) {
            // user did accept the TOS, you can continue interacting with Scoreloop
        }
    }
 });

This call will show the dialog only if neccessary, and resume with the Continuation if the user has accepted.

To check if the user has previously rejected the TOS in order to prevent the user from accessing Scoreloop-depending parts of your game you can call:

 boolean rejected = ScoreloopManagerSingleton.get().userRejectedTermsOfService(null);

If this call returns false, you could disable any Scoreloop functionality. You can also pass a Continuation that will be invoked in case the status changes later:

 ScoreloopManagerSingleton.get().userRejectedTermsOfService(new Continuation<Boolean>() {
    public void withValue(final Boolean value, final Exception error) {
        if (value != null) {
            // value has changed...
        }
    }
 });
