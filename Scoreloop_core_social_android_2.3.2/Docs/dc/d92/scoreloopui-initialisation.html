Initialization and Destruction

The ScoreloopManagerSingleton class is used for the initialization and the destruction of the ScoreloopUI.

Some points to note:

  • The initialization call and the destruction call should happen from within the Android application class and not from within any activity class. Do not forgot to include the name of your application class in the "application" tag of the Android manifest.
  • Initialization and destruction of ScoreloopUI should only happen once in the lifecycle of the game.

Initialization

To initialize ScoreloopUI you must call the ScoreloopManagerSingleton.init(final Context, String) . You need to pass your game secret which you obtain when you register your game with Scoreloop.

Simply add the following code to your Android application class:

// replace "whatever_your_game_secret_is" with your own game secret
ScoreloopManagerSingleton.init(this, "whatever_your_game_secret_is");

Destruction

To destroy the ScoreloopUI you must call the ScoreloopManagerSingleton.destroy() .

Simply add the following code to your Android application class: 

 ScoreloopManagerSingleton.destroy();

Sample Usage

Please find below a typical sample usage example which shows where you could place the init and destroy calls.

 public class myApplication extends android.app.Application {
 
        @Override
        public void onCreate() {
                super.onCreate();
                // replace "whatever_your_game_secret_is" with your own game secret
                ScoreloopManagerSingleton.init(this, "whatever_your_game_secret_is");
        }
 
        @Override
        public void onTerminate() {
                super.onTerminate();
                ScoreloopManagerSingleton.destroy();
        }
 
 }
