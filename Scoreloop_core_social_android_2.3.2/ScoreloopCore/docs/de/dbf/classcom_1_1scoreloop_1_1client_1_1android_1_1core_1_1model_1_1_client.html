Detailed Description

Instances of the Client class set up the underlying infrastructure necessary for using Scoreloop in a game. Initialization should happen as early as possible. Typically only a single instance of Client will be created, which should persist for the entire lifetime of the game.

The class provides accessor methods for setting and retrieving the number of modes and levels in a game, as well as a method for accessing the current Scoreloop session.

Developers must register their games on https://developer.scoreloop.com in order to obtain an identifier and secret for the game application. Both the game-id and game secret are required to initialize an instance of Client.

Member Function Documentation

Range getGameLevels ( )

Accessor that returns the range of levels in a game. Levels are optional game properties that refer to different game stages, (Level 1, Level 2, etc). The range will usually start with 1.

If a game does not support levels, the method will return a range of (0,1) indicating that the game consists of one level.

Returns:
Range The range of game levels.
Range getGameModes ( )

Accessor that returns the range of modes in a game. Modes are optional game properies that refer to different gameplay settings such as speed or difficulty, (easy, medium, hard, etc). Scoreloop produces separate leaderboards for each different game mode. If a game has modes, this method will return the range of modes in the game. If the game does not support modes, the method will return a range of (0,1) indicating that the game consists of a single mode.

Returns:
Range The range of game modes.

void setGameLevels ( final Range  aRange)

Sets the range of levels in a game. Levels are optional game properties that refer to different game stages, (Level 1, Level 2, etc). Levels should also be configured on the Scoreloop developer website, https://developer.scoreloop.com.

If the game does not support levels, the range of game levels does not need to be set.
