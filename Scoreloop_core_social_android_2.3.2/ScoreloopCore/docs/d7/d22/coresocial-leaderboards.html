Scores and Leaderboards

Q. What is a score?
Broadly speaking a score is the result obtained by a player of your game. It can be a simple, single number, or something more complex with several different components.

Q. What are the different score components available to me?
A Scoreloop score can have the following individual components:

  • the score result, (the main result),
  • a minor result, (an optional secondary result),
  • the level, (the game-stage at which the score was achieved),
  • the mode, (the gameplay setting at which the score was achieved).

Q. What are the different score components used for?
The result, minor result, and level can be used for score comparison purposes. The mode - which could refer to things like different difficulty or speed settings - is used in the generation of leaderboards, (Scoreloop generates separate leaderboards for each mode in a game).

Q. Can I define my own score comparison schema?
Yes. You can configure how scores in your game will be compared by visiting https://developer.scoreloop.com. The result, minor result, and level can be configured according to what best suits your game. The default ordering used by Scoreloop is: 

result > minor result> level

You are free to change the ordering of this comparison on the developer website. You can also exclude up to two of the score components from the comparison schema, if you wish, as well as defining whether scores should be sorted in ascending or descending order.

Q. What is the difference between a mode and a level?
Modes refer to different gameplay settings. You might define three modes in your game that represent the settings 'easy', 'normal', and 'difficult', for example. A level refers to the game stage that a score was achieved at, (Level 1, Level 2, Level 3, etc). Modes are used in the generation of leaderboards, levels can be used for score comparison purposes.

Q. What is a leaderboard?
A leaderboard is a sorted list of the best scores submitted by players of your game. A user may only appear once on the same leaderboard - if a user submits a better score than their last submitted score, the better score will replace the existing score and the existing score will be removed from the board.

Q. How many types of leaderboards are there?
Scoreloop can generate the following standard leaderboards:

  • the global leaderboard, (best scores submitted by all players of your game),
  • the 24 hour leaderboard, (best scores submitted in the previous 24 hours),
  • a leaderboard based on the user's nationality,
  • a leaderboard based on the user's geographical location,
  • a leaderboard based on the user's buddy-list.

Scoreloop can generate each of the standard leaderboards for each mode defined in the game, (separate 24h leaderboards for 'easy', 'normal', and 'difficult' modes, for example). Although a single user may only appear once on the same leaderboard, they may appear on separate leaderboards simultaneously, (i.e. on both the global and on the 24 hour leaderboards).

Q. Is there a limit to how many modes I can have in my game?
The only limit imposed is the maximum value set for a java.lang.Integer object.

Q. What classes are involved in the management of scores and leaderboards?
The result obtained by the user in the game should be modelled using an instance of the Score class. Score objects are managed by two data controllers:

  • The ScoreController manages single instances of the Score class and is used to:
    • submit the score to Scoreloop.
  • The ScoresController manages lists of Score objects and is used to:
    • request leaderboards from the server.

An additional class is used to determine the type of leaderboard that is requested from Scoreloop: the SearchList. Before making a server request, the searchlist should be specified in the ScoresController by calling ScoresController.setSearchList(final SearchList).

Q. What is the difference between a ranking and a score?
The Score class models the result obtained in a game. The Ranking class models the position of either a score OR a user on a particular leaderboard, (1st, 2nd, 3rd, etc). To retrieve the ranking for a particular score or user you should use a RankingController should be used. See the accompanying API documentation for more details.

Q. Can I delete scores from a leaderboard?
While your game is still in development you can delete test scores from leaderboards by using the web interface at https://developer.scoreloop.com. Once the game is live it is not possible to delete scores from a leaderboard. If you wish to remove a score from a leaderboard on the basis that it is fraudulent, you should contact Scoreloop via the support forum: https://support.scoreloop.com.

Q. Can I associate some additional game related data like a text string with a Score object ?
Yes, you can associate additional game related data with the score object using the Score Context. There are some methods like setContext() and getContext() available for this purpose.

Q. Is there a limit on how much data can be stored in score context associated with a Score object ?
We don't impose any hard limits on how much data can be stored in the user or score contexts. The only thing to bear in mind with your design is to avoid sending large amounts of data within the score context so that it does not affect the optimal generation of Leaderboards due to the large size of score objects.

Q. Does the Score context get downloaded whenever a Score object is downloaded or is it just retrieved when Score.getContext() is called ?
Yes, at the moment if a Score object has an associated context then it is always present when the object is retrieved from the server, (i.e. Score.getContext() is a local call).
