Social Messaging

The following table below gives an overview of the classes (including the purpose) associated with "Social Messaging".

ClassPurpose
SocialProvider
  • Models a Social Network
SocialProviderController
  • Connect to a social network
MessageController
  • Post Messages to a Social Network

Basic Integration

The following code gives an example of how one might go about connecting to a social provider and posting message.

Use case: Posting a message to a social network

  1. Get an instance of SocialProvider.
  2. Get an instance of SocialProviderController.
  3. Use the provider to check if the user is already connected. If the user is not already connected, use the controller to connect.
  4. Get an instance of MessageController.
  5. Use the MessageController to set the message target, (that is whether it relates to a Scoreloop Score, Challenge, or Game).
  6. Add a receiver to the MessageController. The receiver could be the social provider in question.

Separate instances SocialProvider should be created for each social network that the user will post to.

 //Get instances of SocialProvider
 SocialProvider _myspaceProvider = SocialProvider.getSocialProviderForIdentifier(MySpaceSocialProvider.IDENTIFIER);
 
 SocialProvider _facebookProvider = SocialProvider.getSocialProviderForIdentifier(FacebookSocialProvider.IDENTIFIER);
 
 SocialProvider _twitterProvider = SocialProvider.getSocialProviderForIdentifier(TwitterSocialProvider.IDENTIFIER);

Separate instances of SocialProviderController should be created for each SocialProvider object. The SocialProviderController is used to connect to the social network. 

 //Get instances of SocialProviderController
 SocialProviderController myspaceController = SocialProviderController.getSocialProviderController(mySession, new mySocialProviderControllerObserver(), _myspaceProvider);
 
 SocialProviderController facebookController = SocialProviderController.getSocialProviderController(mySession, new mySocialProviderControllerObserver(), _facebookProvider);
 
 SocialProviderController twitterController = SocialProviderController.getSocialProviderController(mySession, new mySocialProviderControllerObserver(), _twitterProvider);
 
 //Check if the session user is connected.
 if(!_myspaceProvider.isConnected(mySesion.getUser()){
 
 //If not, connect.
 myspaceController.connect(myAndroidGameActivity);
 }
 
 if(!_facebookProvider.isConnected(mySession).getUser()){
 
 facebookController.connect(myAndroidGameActivity);
 }
 
 if(!_twitterProvider.isConnected(mySession.getUser()){
 
 twitterController.connect(myAndroidGameActivity);
 }

Create an instance of MessageController and set its target.

 //Create the MessageController instance
 MessageController myMessageController = new MessageController(new MyMessageControllerObserver());
 
 //Set the target (in this case it will be the current game)
 myMessageController.setTarget(mySession.getGame());

Before the MessageController can be used to post messages relating to the target, the correct receivers must first be added. The receivers in the example below are the social provider instances.

 //Add receivers. 
 //(the below snippet is to post messages to the user's "wall").
 // Refer to the api addReceiverWithUsers() to post messages to a specific list of users.
 
 myMessageController.addReceiverWithUsers(_myspaceProvider);
 myMessageController.addReceiverWithUsers(_facebookProvider);
 myMessageController.addReceiverWithUsers(_twitterProvider);
 
 //A sample message.
 String message = "I just joined Scoreloop!";
 
 //Post the message.
 myMessageController.setText(message);
 myMessageController.submitMessage();

The message that will get posted with the above code will be something similar to "Player2313 is playing SampleGame: I just joined Scoreloop!".
