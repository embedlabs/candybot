News

The following table below gives an overview of the classes (including the purpose) associated with "News".

ClassPurpose
RSSItem
  • Models a feed item.
RSSFeed
  • Allows to retrieve feed items from the Scoreloop server to be displayed in your application
RSSFeedObserver
  • Callback interface used by a RSSFeed

Basic Integration

The following example shows how you could go about integrating news feed into your application.

  1. Create an RSSFeed Object by calling RSSFeed() . You will need to pass a RSSFeedObserver as an argument to the constructor in order to receive callbacks.
  2. Call the method requestNextItem() to load the latest feed from the server. You will have to pass a policy which suits your criteria on how you would like to filter out the RSSFeed Items. You could use the pre-defined policies STANDARD_POLICY, STICKY_POLICY, UNREAD_POLICY or ON_START_POLICY. You also have the option of defining your own policy. See Policy and ChainedPolicy for more details.
  3. Await server response via observer callback methods. See RSSFeedObserver for more details.
  4. Display the received feedItem in the UI.
  5. Go to Step 2 and repeat 2,3,4 till there are no more Items returned.
  6. Call reloadOnNextRequest() and then go to Step 2.
See also:
RSSFeed , RSSFeedObserver , RSSItem , ChainedPolicy , Policy.
 RSSFeed rss_feed;
 
 // assumes observer - RSSFeedObserver exists or pass null. 
 rss_feed = new RSSFeed(observer);
 
 rss_feed.requestNextItem(STANDARD_POLICY);

Await the successful call via the delegate callback methods, such as:

 public void feedDidReceiveNextItem (RSSFeed rss_feed, RSSItem rss_item) {
 
        //rss_item is the received RSSItem
        if (rrs_item)
        {
                // Display the RSSItem in the UI 
        }
        else //rss_item is null
        {
                // No more items are returned 
                rss_feed.reloadOnNextRequest();
        }
 }
