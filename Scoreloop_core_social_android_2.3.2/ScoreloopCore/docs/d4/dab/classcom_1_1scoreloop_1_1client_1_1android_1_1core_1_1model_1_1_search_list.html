Detailed Description

A SearchList is an abstraction for filtering and sorting a set of entities that are stored on the Scoreloop servers. Searchlists are used by the following controllers:

  • ScoresController
  • RankingController

to filter scores and rankings respectively.

SearchLists can be set up to filter entities according to:

  • global search parameters,
  • the buddy-list of a user,
  • only entities that have been updated in the previous 24 hours,
  • the user's location,
  • the user's nationality.

The global search list is also the default search list.

Basic Usage:

  1. Set the SearchList of the relevant controller to perform the required filtering.
    For example, to filter scores according to the buddy-list of user, set the SearchList of the ScoresController as follows:
    • ScoresController.setSearchList(getBuddiesScoreSearchList()).
  2. Use the controller to make the server request as normal.

static SearchList getUserNationalityScoreSearchList ( ) [static]

Returns a searchlist based on the nationality of the user. Unlike the searchlist returned by getUserCountryLocationScoreSearchList(), the searchlist returned here will not change based on the user's location.

Nationalities are based on the country codes defined by the ISO-3166 standard.
