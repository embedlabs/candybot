Detailed Description

The RequestControllerException is delivered via the RequestControllerObserver.requestControllerDidFail(RequestController, Exception) callback.

Basic Usage:

  1. Cast the Exception argument from the RequestControllerObserver.requestControllerDidFail(RequestController, Exception) callback as a RequestControllerException.
  2. Use getErrorCode() to access the error code.

Many of the error codes are self-explanatory. However, if the undefined error code (=0) is returned you may find additional useful information via the error detail. To access this information:

  1. Call the hasDetail(final int) method.
  2. Pass each of the error detail constants as arguments, (these are of the form DETAIL_XXXX_XXXX).
  3. Note whether the method returns true or false.

If getErrorCode() returns CODE_SOCIAL_PROVIDER_DISCONNECTED you can find out more information about which social network has disconnected by doing the following:

  1. Call getUserInfo() to return a java.util.Map object.
  2. Use the INFO_KEY_DISCONNECTED_PROVIDER_IDENTIFIERS constant as a key in this map.
  3. Note the additional information about what social network disconnected.

Map<String, Object> getUserInfo ( )

This method returns additional meta-data about the user. This is particularly useful if the getErrorCode() method returns CODE_SOCIAL_PROVIDER_DISCONNECTED, as the java.util.Map that is returned here can provide additional inforamtion about what social network disconnected. You should:

  1. Call getUserInfo() to access the java.util.Map.
  2. Use the INFO_KEY_DISCONNECTED_PROVIDER_IDENTIFIERS constant as a key in this map to access additional information about the error.

Member Data Documentation

final int CODE_BUDDY_ADD_REQUEST_ALREADY_ADDED = 40 [static]

This code is returned if the user attempts to add someone who is an existing buddy to their buddy-list.

final int CODE_BUDDY_REMOVE_REQUEST_ALREADY_REMOVED = 41 [static]

This code is returned if the user attempts to remove an already removed buddy, from their buddy-list.

final int CODE_SOCIAL_PROVIDER_DISCONNECTED = 110 [static]

This code is returned if application disconnects from from a social network.

final int CODE_UNDEFINED = 0 [static]

This code is returned if the error is undefined elsewhere in the class. If you receive this code you should use the hasDetail(final int) method to find out more information.

final int DETAIL_USER_UPDATE_REQUEST_EMAIL_TAKEN = 1 << 4 [static]

Pass this constant to the hasDetails(final int) method to establish if the exception is due to the user attempting to update their user profile with the email address of an existing Scoreloop user.

final int DETAIL_USER_UPDATE_REQUEST_IMAGE_TOO_LARGE = 1 << 5 [static]

Pass this constant to the hasDetails(final int) method to establish if the exception is due to the user attempting to update their profile image with an image that exceeds the permitted size.

final int DETAIL_USER_UPDATE_REQUEST_INVALID_EMAIL = 1 << 3 [static]

Pass this constant to the hasDetails(final int) method to establish if the exception is due to the user attempting to update their profile with an invalid email address.

final int DETAIL_USER_UPDATE_REQUEST_INVALID_USERNAME = 1 << 0 [static]

Pass this constant to the hasDetails(final int) method to establish if the exception is due to the user attempting to update their profile with an invalid username, (with prohibited characters for example).

final int DETAIL_USER_UPDATE_REQUEST_UNSUPPORTED_MIME_TYPE = 1 << 6 [static]

Pass this constant to the hasDetails(final int) method to establish if the exception is due to the user update request using an unsupported MIME type.

final int DETAIL_USER_UPDATE_REQUEST_USERNAME_TAKEN = 1 << 2 [static]

Pass this constant to the hasDetails(final int) method to establish if the exception is due to the user attempting to update their profile with the username of an existing Scoreloop user.

final int DETAIL_USER_UPDATE_REQUEST_USERNAME_TOO_SHORT = 1 << 1 [static]

Pass this constant to the hasDetails(final int) method to establish if the exception is due to the user attempting to update their profile with a username that is shorter than the permitted minimum. Usernames must be at least 3 characters long.

final String INFO_KEY_DISCONNECTED_PROVIDER_IDENTIFIERS = "disconnectedProviderIdentifiers" [static]

If the error code returned is equal to CODE_SOCIAL_PROVIDER_DISCONNECTED do the following to find out mode details:

  1. Call the getUserInfo() method to access the java.util.Map<String, Object>.
  2. Use this constant as a key in that map to find out more information about which social network disconnected.
