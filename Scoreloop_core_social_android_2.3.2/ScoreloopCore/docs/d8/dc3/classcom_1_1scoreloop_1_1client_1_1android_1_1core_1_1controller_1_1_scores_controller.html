Detailed Description

The ScoresController retrieves lists of Score objects from the server.
Basic usage:

  1. Set the search list with setSearchList().
  2. Set the mode for the controller with setMode().
  3. Set the number of scores that will be retrieved with setRangeLength().
  4. Request the scores using one of the load methods:
    • loadRangeAtRank()
    • loadRangeForScore()
    • loadRangeForScoreResult()
    • loadRangeForUser()

When the request is done, (see RequestControllerObserver), access the scores list using getScores().

To page through additional scores:

  1. Use hasNextRange() and hasPreviousRange() to determine if it is possible to page.
  2. Use loadNextRange() or loadPreviousRange() to request the next or previous list of scores from the server.
  3. Use getScores() to access the new list, as above.

At any time it is possible to set up the controller for a completely different search. Score sorting is done server side. The score comparison schema is defined by the developer and can be modified at https://developer.scoreloop.com.

List<Score> getScores ( )

Accessor that returns the list of Score objects that have been retrieved from the server by calling one of the load methods. The desired size of the list is defined using setRangeLength(int), but the actual size might be smaller. Call size() on the returned list to query the actual size.

void loadRangeAtRank ( final int  rank)

Requests the list beginning at the given rank. The rank refers to a score's position on the global leaderboard. All lists start at 1, therefore the minimum rank possible is 1.

void loadRangeForScore ( final Score  aScore)

Requests the list of scores that includes the given Score in the middle. The score is included even if it did not make it in the high score list.
