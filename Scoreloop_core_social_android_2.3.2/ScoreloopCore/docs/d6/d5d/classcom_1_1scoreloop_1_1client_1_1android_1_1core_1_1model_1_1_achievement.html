Detailed Description

The Achievement class models whether an Award has be achieved by a user or not. You query this state by testing the isAchieved() property. If it equals true, then the award has been achieved by the user, if it equals false then it has not.

To achieve an award a user must reach an arbitrary value set by the developer, (this is 1 in most cases). This "@link #com.scoreloop.client.android.core.model.Award.getAchievingValue() achieving value@endlink" can be reached in stages, if desired. Both the "achieving value" of an Award and the number of stages it takes to reach it must be configured by the developer on https://developer.scoreloop.com. After awards are configured on the developer website, Scoreloop generates an awards bundle, (sl_awards_bundle.zip), that must be downloaded and added to your game project. This bundle provides local access to awards and achievements on the client device.

You can mark an Achievement as being unlocked in one of two ways:

  • In a single step with a call to the method setAchieved().
  • Or by incrementing a value in stages until it equals the " @link #com.scoreloop.client.android.core.model.Award.getAchievingValue() achieving value@endlink" for the award.

As an example, a developer might configure an award so that the "achieving value" equals 10, and that this may be reached in 10 separate incremental stages. In the gameplay, this might mean that the user could achieve the award by collecting 10 secret jewels - finding a single jewel would represent completing one of the configured incremental stages. After the tenth jewel has been found then the user has reached the configured "achieving value" and achieves the award.

After an award is achieved by one of the two approaches mentioned above, you have to inform Scoreloop by submitting the relevant Achievement objects with the help of an AchievementController.

To retrieve Achievement objects in the first place, use an AchievementsController.

Date getDate ( )

This method returns the date on which the award was achieved. It will have a value of null if the award has not yet been achieved.

Returns:
java.util.Date, the date on which the Award was achieved
Bitmap getImage ( )

The image associated with the Achievement. The exact image returned by this method is dependent on the value returned by isAchieved():

  • if it returns true the image accessed via Award.getAchievedImage() will be returned,
  • if it returns false the image accessed via Award.getUnachievedImage() will be returned.

The different award images can be configured on https://developer.scoreloop.com.
If the developer does not upload any custom images, then default stock Scoreloop images will be returned instead.

int getValue ( )

For an award to be achieved by a user, they must reach an arbitrary value set by the developer that has been configured on https://developer.scoreloop.com. This arbitrary value, (accessed via Award.getAchievingValue()) may be reached in incremental stages, (depending on how the award has been configured).
This number of incremental stages can be accessed via Award.getCounterRange().
This method returns a value representing the "position" of the user within this range of incremental stages.

The award will be marked as achieved when the value returned by getValue() equals that returned by Award.getAchievingValue().

Returns:
int, An integer value within the Award.getCounterRange() of the award associated with this Achievement.
boolean isAchieved ( )

A predicate method used to query whether the Award associated with this Achievement has been achieved by the user. This method will return true if the value returned by getValue() is greater or equal to Award.getAchievingValue().

Returns:
boolean, true if the award has been achieved by the user, otherwise false.
boolean needsSubmit ( )

Predicate method used to query whether the Achievement needs to be submitted to Scoreloop. This method will return true when the value returned by isAchieved() is also true.
If the Achievement needs to be submitted to Scoreloop, you will need to use an instance of AchievementController to make the submission.

Returns:
boolean Will return true if the Achievement should be submitted to Scoreloop, otherwise false.
void setAchieved ( )

This method explicitly sets the status of the Award associated with this Achievement as having been "achieved" by the user. After this method has been called:

  • getValue() will equal Award.getAchievingValue(),
  • needsSubmit() will return true.

void setValue ( final int  value)

For an award to be achieved by a user, they must reach an arbitrary value set by the developer that has been configured on https://developer.scoreloop.com. This arbitrary value, (accessed via Award.getAchievingValue()) may be reached in incremental stages, (depending on how the award has been configured).
This number of incremental stages can be accessed via Award.getCounterRange().

This method can be used to increment the "position" of the user within this range of stages, by setting the stage equal to the value passed as an argument.
If the value passed as an argument is not a valid parameter, (that is, if it does not fall within the Award.getCounterRange()), a java.lang.IllegalArgumentException will be thrown.
If the value passed as an argument is greater or equal than Award.getAchievingValue() then the award will be marked as achieved for the user and:

  • isAchieved() will return true,
  • needsSubmit() will return true.
Parameters:
valueThe value to be set.
