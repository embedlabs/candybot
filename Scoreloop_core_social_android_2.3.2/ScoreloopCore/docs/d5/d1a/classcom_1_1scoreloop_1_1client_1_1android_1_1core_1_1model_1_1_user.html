Detailed Description

The User class models the game user. Scoreloop considers users to be unique players tied to unique devices. The Session class maintains an instance of the local User (referred to as the session user).

User objects that are not the session user may not have all properties available.

However:

  • the identifier, login, and display name properties will always be available.
  • whether a User is challengable becomes available when the User comes from a ScoresController.
  • all properties except balance, the challenge stakes, and authenticated become available after using a UserController to loadUserDetail()

User objects are returned in the following scenarios:

  1. From the Scoreloop Session, (the session user).
  2. From other User objects, (from another user's buddy-list).
  3. From a Score object, (the user who submitted the score).
  4. From a UserController, (the user set for the controller).
  5. From a UsersController, (the list of users) the controller retrieves from the server).
  6. From a Challenge object, (the challenge contender, contestant, winner, and loser).

Basic Usage:

User objects are managed by two controllers:

  1. The UserController which is used to retrieve information for an arbitrary user from the server. It is also used to update the attributes of the session user.
  2. The UsersController, which retrieves lists of User objects from the server.
See also:
UserController, UsersController, Guide to Users

Integer getBuddiesCounter ( )

This method returns the number of Scoreloop users in the buddy-list of a given User instance.

Returns:

List<User> getBuddyUsers ( )

Accessor that returns a list of User objects representing the Scoreloop buddies of the receiver.

Use a UserController to call loadBuddies() before this method is called, otherwise this method will return an empty list.

Returns:
java.util.List of User objects. The user's buddy-list. This will return null if the user has not added any buddies yet.

boolean isAnonymous ( )

Examines the state of the user to see if it is anonymous. When a device connects to Scoreloop, if no user information can be found for it, a username is automatically generated for the device and the state of the user set to "anonymous".

Anonymous users can upload high-scores, but not challenges.

This property is available to the session user.

Returns:
true if the user's state is "anonymous", otherwise false
boolean isAuthenticated ( )

Accessor to check whether the user authentication on the server succeeded. This property is available to the session user.

Returns:
true if the user has been authenticated, otherwise false.

boolean isPassive ( )

Examines the state of the user to see if it is passive. A "passive" state describes an anonymous user who has chosen a username and who is identified by their device.
