Detailed Description

The Scoreloop Session.

The Session class models the session between the game application and the Scoreloop servers. An instance of Session is created transparently when the Client object is instantiated.

The following properties and entities are accessible through the Session object:

  • The session User.
  • The balance of the session User.
  • The Challenge currently underway (if any).
  • The permitted challenge stakes.
  • The Game object.
  • The list of available SearchList objects.
  • The "authenticated" state of the Session.
  • The Payment URL
  • The Scoreloop App Download URL

The Session will read any existing data for the session user from secure storage, if available.

The Session will be authenticated after the first request is made to the server.

Challenge getChallenge ( )

Accessor that returns the Challenge currently underway in the Session, (if any).

The Challenge object will be set in and removed from the Session automatically by a ChallengeController.

Alternatively, challenges can be set in the Session manually by calling setChallenge(Challenge).

When you submit a score via the com.scoreloop.client.android.core.controller.ScoreController#submitScore() and shouldSubmitScoreForChallenge returns true, then the score will be submitted as part of the challenge object returned by this method.

Returns:
Challenge The challenge currently underway in the session. Will return null if no challenge is underway.
List<Money> getChallengeStakes ( )

Accessor that returns the permitted stake amounts available for the user to use in challenges.

The lowest stake amount permitted is dependent on the balance of the current session user, and will generally increase as the user's balance increases.

Users with low balances may use relatively small stakes when instigating challenges. Users with higher balances will have to use slightly higher stakes when starting challenges.

Returns:
java.util.List of Money objects modelling the challenge stakes available to the current session user.

String getPaymentUrl ( )

This method is used to return the payment URL. The payment URL is automatically created during the session handshake with Scoreloop. Using this payment URL, game players can buy 'coin packs' using their credit cards. For more details on how to configure and purchase coin packs for Android Games, please have a look at our FAQ Purchasing Coin Packs for Android Games

Returns:
java.lang.String, the string representation of the URL of the payment web page.
