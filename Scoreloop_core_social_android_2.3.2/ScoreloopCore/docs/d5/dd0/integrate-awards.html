Awards and Achievements

The following table below gives an overview of the classes (including the purpose) associated with "Awards and Achievements".

ClassPurpose
Award
  • Model class used to store the award information configured by the developer on the Scoreloop Developer website
Achievement
  • Model class used to model whether an award has been achieved by a user or not, and as such it links a specific Award object with a boolean value corresponding to its status, (i.e. "achieved" or "not achieved").
AchievementController
  • Manages single instances of the Achievement class
AchievementsController
  • Retrieve lists of achievements from Scoreloop servers

Basic Integration

Integrating Awards and Achievements into your game consists of the following steps:

  1. Configure the game awards on the Scoreloop developer website https://developer.scoreloop.com.
  2. Adding the awards bundle to the game project.
  3. Retrieve list of Achievements (which report whether an Award is achieved or not) form the Scoreloop server using the AchievementsController.
  4. Record an achievement made by a user.
  5. Use the AchievementController to submit an achieved award to the server.

1. Configuring awards on Scoreloop Developer website

Configure your game awards by navigating to the Awards section of your game page on the Developer website. The configuration process includes choosing a unique reverse DNS identifier for every award, specifying the “achieving value” of an award and the number of stages required to reach it etc.

2. Adding the awards bundle to the game project

Once you have configured awards on the Scoreloop Developer website, you need to:

  1. download and uncompress the zip file (resulting in a folder named SLAwards.bundle) that Scoreloop generates and
  2. add the awards bundle generated by Scoreloop to the assets folder of your game project.

This provides local access to information about the configured awards, including their identifiers and achieving values, among other things. If you make any changes to to your awards on the Developer website you must download the new bundle from the site and update your assets directory so that any changes are reflected in your game.

3. Retrieve the list of Achievements

Retrieving the list of Achievements consists of the following steps:

  1. Get an instance of AchievementsController or re-use an existing one.
  2. Achievements for the session user will be retrieved by default. If achievements for a user other than the session user are to be retrieved, then set the user by calling the setUser() method.
  3. Call AchievementsController's loadAchievements() method to make the request.
  4. Await notification of success via delegate callbacks.
  5. After a successful request, use getAchievements() method to access the retrieved Achievements array.
 {
 ...
 
 // get an instance of AchievementsController
 myAchievementsController = new AchievementsController(new AchievementsObserver());
 
 // optionally set the user if you want to retrieve Achievements for a user other than the session user
 myAchievementsController.setUser(someUser);
 
 // make request to the server to load the Achievements
 myAchievementsController.loadAchievements();
 }
 
 // observer callback for success case
 public void requestControllerDidReceiveResponse(RequestController aRequestController) 
 {
 // acces the Achievements Array
 List<Achievements> = myAchievementsController.getAchievements();
 
 // additional code here
 ...
 }

4. Record an achievement made by a user

When the user has reached a game-state which justifies the unlocking (achieving) of an Award, he needs to set the achieved property of the Achievement object. The steps involved are:

  1. Optionally check whether an award has reached the achieving value by calling the method isAchieved() . This method will return true if the award has already been achieved.
  2. Either call setAchieved() or explicity set the value of the Achievement via setValue() .
 // optionally, check whether an award has reached the achieving value
 if (!myAchievement.isAchieved()) {
 
 // set the achievement as being achieved 
 myAchievement.setAchieved();
 
 // or alternatively, set the value of the Achievement
 myAchievement.setValue(myValue); 
 
 ...

5. Use the AchievementController to submit an achieved award to the server.

To submit an Achievement to the server:

  1. Check with the needsSubmit() method on an Achievement object whether the Achievement needs to be submitted to the server.
  2. If so, create an instance of AchievementController or re-use an existing one.
  3. Call AchievementController's setAchievement() method to set the Achievement.
  4. Call the AchievementController's submitAchievement() method.
  5. Await notification of success via observer callbacks.
 // checking if the Achievement needs to be submitted or not
 if (myAchievement.needsSubmit()) {
 
 // create an instance of Achievement Controller - or reuse an existing one
 myAchievementController = new AchievementController(new AchievementObserver());
 
 // set the achievement on the contorller
 myAchievementController.setAchievement(achievement);
 
 // submit the achievement to the server
 myAchievementController.submitAchievement();
 }
 
 // observer callback for success case
 public void requestControllerDidReceiveResponse(RequestController aRequestController) 
 {
 // optionally check and submit other achievements which need to be submitted 
 ...
 }
