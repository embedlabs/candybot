Detailed Description

The MessageController is used to post messages to a set of social networks, (or other receivers). Posted messages usually refer to some Scoreloop-related event such as winning a challenge or posting a score.

Basic Usage:

To post a message to a social provider:

  1. First check if the user is already connected to a social provider. Use the isUserConnected(User) of the appropriate SocialProvider subclass to do this.
  2. If the user is not connected, connect with an instance of SocialProviderController.
  3. Once connected, create an instance of MessageController and set the message target to the object causing the message, (setTarget(Object)).
  4. Use addReceiverWithUsers(Object, List<User>) to add at least one receiver to the controller with (optionally) a list of users. Note that when a social provider is added as a receiver it must be in a a connected state. See 1 and 2 above.
  5. Call submitMessage() to post to the relevant social network(s).
See also:
RequestControllerObserver, SocialProviderController, SocialProvider

Member Function Documentation

void addReceiverWithUsers ( final Object  receiver,
final Object...  users 
)

This method adds a receiver and a list of users to the controller. Valid receivers are currently:

  • any supported SocialProvider SocialProvider
  • the e-mail receiver identifier RECEIVER_EMAIL
  • the system receiver identifier RECEIVER_SYSTEM
  • the user receiver identifier RECEIVER_USER

Important:

Ensure that the social provider instance is in a connected state before adding it as a receiver.

Parameters:
receiverThe receiver to be added to the controller. See above for the list of accepted receiver types.
usersjava.util.List of users that will receive the message.

For social provider receivers, don't pass any values in order to post the message to the session-users "wall" or homepage.

For email receivers, this parameter must contain at least one valid email address string.

For system receivers you must not pass any parameters.

For user receivers, the parameter objects (at least one) must be valid instances of the User class.

Note: For email receivers, this parameter must contain at least one valid email address string.

Exceptions:
Raisesan java.lang.IllegalArgumentException if some problem is detected with any of the parameters.

Object getTarget ( )

Method that returns the target the message refers to. Objects will be one of:

  • Challenge
  • Score
  • Game
  • Achievement
  • User
  • INVITATION_TARGET

Note, that only objects with a valid Scoreloop identifier property may be posted as messages. Use the appropriate Scoreloop controller to submit the target objects to the Scoreloop servers first. This will result in the identifier property being set.

boolean isSubmitAllowed ( )

Assesses whether a message can be posted or not. Reasons for not posting include: the user not being connected to the social provider receiver, or if the receiver list is empty, or if the target is null..

Returns:
true if the message can be posted, otherwise false.
void setMessageType ( final int  messageType)

Method to set the message type to something other than TYPE_TARGET_INFERRED. By default the message type is TYPE_TARGET_INFERRED. E.g. when you want to report an abuse related to some user, set the type to TYPE_ABUSE_REPORT.

Note: When the message type is set to TYPE_ABUSE_REPORT, make sure that the Receiver set is RECEIVER_SYSTEM, and the target is User. When the message type is set to TYPE_RECOMMENDATION, make sure that the Receiver set is RECEIVER_USER, and the target is Game.

Parameters:
messageTypecurrently either TYPE_TARGET_INFERRED, TYPE_ABUSE_REPORT or TYPE_RECOMMENDATION

Posts the message. This method will first verify that the message properties are in order and that it may be posted, before requesting the Scoreloop servers to post the message to the specified receivers.

Member Data Documentation

final Object INVITATION_TARGET = new InvitationTarget() [static]

Use this object as the message target when you want to invite other users. Currently only E-Mail receivers are allowed together with this target. So, use addReceiverWithUsers() with a first argument of RECEIVER_EMAIL and provide the email address of the person you want to invite as second argument.

final Object RECEIVER_EMAIL = new EmailReceiver() [static]

Use this instance when you want to send a message to an e-mail address. When you provide this object as first argument to addReceiverWithUsers(), you have to provide at least one email address list of the second parameter.

final Object RECEIVER_SYSTEM = new SystemReceiver() [static]

Use this receiver when you want to report an abuse to Scoreloop.

final Object RECEIVER_USER = new UserReceiver() [static]

Use this receiver when you want to send messages to another Scoreloop users where their User objects are known.

final int TYPE_ABUSE_REPORT = Message.Type.ABUSE_REPORT.ordinal() [static]

Use this message type to report an abuse.

final int TYPE_RECOMMENDATION = Message.Type.RECOMMENDATION.ordinal() [static]

Use this message type to recommend a game. You have to provide a Game object to setTarget() in this case.

final int TYPE_TARGET_INFERRED = Message.Type.TARGET_INFERRED.ordinal() [static]

An implicit message type which is inferred from the specified target object.
