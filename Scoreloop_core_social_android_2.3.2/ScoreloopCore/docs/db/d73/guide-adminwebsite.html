The Scoreloop Developer Website

All games must be registered and properly configured on the Scoreloop Developer website. If you have not alread done so, please go to https://developer.scoreloop.com and create a developer account. Then you will be able to:

  • register your game and obtain application data for your game
  • configure your game properties
  • configure the scoring schema
  • configure awards
  • configure custom emails

Registering Your Game

This is a necessary first step in using the CoreSocial SDK . By registering your game on the developer website, you receive two pieces of data unique to your game: a game identifier, and a game secret.

Both the game identifier and the secret are must be added to your game code. It is therefore important that you first register your game and obtain both before starting to implement CoreSocial.

The game identifier and the secret can be retrieved at any time by logging onto https://developer.scoreloop.com. In addition, you should also download scoreloop.properties file from the developer website and add it to your assets folder.

Each game that you wish to integrate with CoreSocial must be registered separately. However, you need only create a single Scoreloop account from which you can have instant access to all of your registered games.

Configuring Your Game Properties

All registered games must contain (minimally) a name. You can also configure additional game properties including:

  • a short description of your game
  • publisher's name
  • purchasing information
  • the current state of the game (e.g. "in development" or "live")

The name and additional descriptive information you give your game can be changed subsequently through the edit capabilities of the Developer website. The game identifier and game secret that were generated by Scoreloop will, however, always remain the same.

The Scoreloop servers are automatically updated to reflect any changes or edits made by you through the developer website.

Configuring the Scoring Schema

You may configure a schema on the Developer website which will be used by Scoreloop to compare scores that are submitted by users of your game. There are three properties which may be configured:

  • Game result, (the primary game result obtained by users of your game)
  • Minor result, (an optional secondary result which may also be considered)
  • Level, (the game level at which the score was achieved, this is also optional)

You can configure an arbitrary comparison order for these three properties, (e.g. first compare the game level, then the result, then the minor result). Up to two properties may be excluded from the schema, (at least one property must remain in order for the comparison to be valid). Each property can be configured so as to be sorted in ascending or descending order.

If you wish you may use Scoreloop's default comparison schema: 

 result > minorResult > level.

Configuring Awards

Your game might support awards which are target states in your game that can be achieved by the user. Achievments are awards unlocked during the gameplay. Developers must configure awards on the Scoreloop developer website. Once an award has been configured on the website, Scoreloop generates an awards bundle, which developers must add to their assets folder of the game project. The awards bundle provides local access to the individual award properties.

Configuring Custom Emails

You may configure custom emails on the developer website. Custom emails are triggered after certain challenge events, such as winning or losing a challenge, or when a challenge expires. You are free to customize these emails by providing two images for display in an email header. The main image will appear only once and will generally include the game name and game icon. The second image will repeat after the main image and should be configured if you wish your email header to have a “banner” effect.
