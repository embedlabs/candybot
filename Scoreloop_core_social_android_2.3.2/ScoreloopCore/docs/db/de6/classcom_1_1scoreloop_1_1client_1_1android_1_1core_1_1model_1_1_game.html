Detailed Description

The Game class models the game application. The class provides accessor methods for retrieving and modifying certain game properties, including:

  • modes (optional properties referring to different aspects of gameplay such as speed or difficulty)
  • levels (optional properties referring to different game stages)
  • the game name
  • the game version

Games must be registered on https://developer.scoreloop.com, where they will be assigned a unique game-id, and where developers can configure additional game properties.

Basic Usage:

Instances of Game are not created directly, but are instead available through the current Session object.

  1. Create an instance of Client using the game-id and game secret from https://developer.scoreloop.com.
  2. An instance of Session is created transparently with each Client object. Use the Client object to call getSession() and retrieve the current session object.
  3. Use the current Session object to call getGame() which will return an instance of Game.
See also:
Client, Session, Guide to Developer Website
