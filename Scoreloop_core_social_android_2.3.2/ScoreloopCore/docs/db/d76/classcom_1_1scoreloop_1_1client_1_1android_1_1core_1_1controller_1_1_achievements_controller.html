Detailed Description

The AchievementsController is used to retrieve the current list of Achievements from the server.

Note that developers must configure awards for their game on https://developer.scoreloop.com. Part of the configuration process includes choosing a unique identifier for every award (e.g. using a reverse DNS scheme). Once awards have been configured, Scoreloop generates an awards bundle, which developers must add to their game's assets folder. This provides local access to information about the configured awards, including their identifiers and achieving values, among other things.

Basic usage:

To retrieve the list of Achievement objects from the server:

  1. Create an instance of an AchievementsController or re-use an existing one.
  2. If achievements for a user other than the session user are to be retrieved, set the user by calling the setUser() method.
  3. Call AchievementsController's loadAchievements() method to make the request.
  4. Await notification of success via the standard observer callbacks.
  5. After a successful request, use the getAchievements() method to access the retrieved Achievements array.

Member Function Documentation

int countAchievedAwards ( )

This convenience method counts how many awards have been achieved by the user and returns the total. The method requires that the achievements array contain actual values. Consequently, the asynchronous method loadAchievements() must be called before countAchievedAwards to ensure that the achievements array has been filled.

Returns:
int, the number of awards achieved by the user.

List<Achievement> getAchievements ( )

This method is used to access the array of Achievements loaded for the user. This method returns all the Achievement objects for the specified user that were retrieved from the server by calling loadAchievements(). Please note that loadAchievements() should be called before calling this method!

Returns:
a list of Achievement instances for the specified user.
AwardList getAwardList ( )

This method returns an array holding all the awards available for the particular game version. The information is read locally from awards bundle that has been generated by Scoreloop and which has been added to the assets folder of the game.

Returns:
AwardList, Array of Award Objects or null if no (or no valid) awards bundle was found.

void loadAchievements ( )

This asynchronous method requests the achievements for the user specified by the controller's user property. If no user has been specified, the achievements for the current session user will be retrieved by default.

After a successful server request, access the retrieved achievements by calling the getAchievements() method,

void setForceInitialSync ( final boolean  forceInitialSync)

Method to set the forceInitialSync parameter which determines whether to initially sync with the Scoreloop Server to load the achievements or just load the achievements from the local db.

By default, the forceInitialSync is set to true (ie there will be an initial sync with the Scoreloop server when calling loadAchievements()) By setting the forceInitialSync to false, you can override the initial server sync when calling loadAchievements() (ie just load the achievements from the local db).

Parameters:
forceInitialSyncfalse if you want to load achievements from the local db, true if you want the initial sync with the server.
