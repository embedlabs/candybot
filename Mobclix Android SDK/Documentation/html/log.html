[Mobclix for Android] >> [Changelog]

Version 4.0.2

  • Bug Fix: Fixed an issue where the Mobclix SDK could crash certain devices running Android 2.1 while managing the LocationManager.
    This issue has been resolved.
  • Bug Fix: Open Allocation Generic now schedules its next ad request without having to call continueRequest().
    The new implementation of ad refreshing wasn't properly implemented on Open Allocation Generic. This has been resolved.
  • Bug Fix: Fixed an occasional crash in MobclixInstrumentation.
    Better error catching has been put in place to fail gracefully.
  • General: Improved resource usage.
    Code has been added to better handle the Mobclix SDK running out of memory when decoding large creatives.

Version 4.0.1

  • Bug Fix: Fixed an issue where the Mobclix SDK was crashing when loading some urls.
    This issue has been resolved.
  • Feature: Added support for the Expand to Fullscreen button for HTML5 inline videos on Honeycomb and ICS.
    Since Honeycomb, WebViews, and in turn our AdView, support inline videos. The button to expand to fullscreen was causing some issues. The button is now supported and will expand the video to fullscreen according to the current orientation of the device.
  • Known Issue: Some instability in certain cases using the Expand to Fullscreen button.
    There is an instability in the Android webkit implementation of HTML5 video and expanding to the full screen button. We've brought this issue up to Google and are working with our Ad Network partners to prevent them from sending poorly constructed creatives.

Version 4.0.0

  • Feature: HTML5 video ad support for ICS devices.
    ICS devices were running into an issue where certain HTML5 video ads were playing in the background before an ad was displayed. This issue has been resolved.
  • Feature: Added MRAID Rich Media ad support.
    Mobclix is now MRAID Version 1 compliant and can serve MRAID Version 1 ads.
  • Feature: Developer debug logging.
    Test devices set on the Mobclix Developer Dashboard will now log certain debug information about creatives for the developer.
  • Feature: Close button for expanded ads.
    A close button has been added to expanded ads similar to the close button found in full screen ads.
  • Bug Fix: Full Screen Ad crash.
    Fixed an issue that would crash when loading a Full Screen Ad before a WebView was created by the app in devices running Android OS 4.0.0+.
  • Bug Fix: Video ads playing too early.
    Fixed an issue in which fullscreen video ads would play before they were displayed on screen.
  • General: Improved resource usage.
    General improvements to the SDK regarding memory management and CPU usage.

Version 3.2.0

  • Mobclix no longer passes personally identifying information unencoded. This should allow apps using Mobclix to be accepted into the Amazon App Store.
  • Improved the ability of a developer to cancel an ad request in progress.
  • Added an error code for when an ad request is canceled.

Version 3.1.4

  • Improved orientation detection in Android 2.2+ devices to support Reverse Portrait and Reverse Landscape orientations.
  • The MobclixAdView no longer has scrollbars when in banner form.
  • Fixed a bug where remote configuration could fail too soon when passed an unsupported ad size.
  • Deprecated some Rich Media methods for Android SDK 1.6 and below.
  • The MobclixFullScreenAdView now prevents more than one FullScreenAdView from displaying at once.
  • Fixed an issue where a preloaded Video ad would cause issues with OpenGL SurfaceViews.
  • Improved memory management.

Version 3.1.3

  • Disabled WebView plugins for Android API level 3+, preventing the Flash ad crash.
  • Added additional checks to prevent leaking impressions.
  • A number of fixes to cause code to crash gracefully.

Version 3.1.2

  • Added a close button to the MobclixFullScreenAdView.
  • Added checks to prevent a MobclixFullScreenAdView from requesting an ad when an ad is already displayed.
  • Moved Open Allocation onAdClick listener triggers into the "MMAdClickedToNewBrowser" and "MMAdClickedToOverlay" listeners of the Millennial Media SDK and the "onPresentScreen" listener of the Google AdMob SDK.

Version 3.1.1

  • Worked around a memory leak in the Google AdMob SDK so that Open Allocation Google/AdMob users are less likely to leak memory.
  • Updated the Detailed Start Guide for Android with details on how to set up Proguard to work with Open Allocation Millennial Media.
  • Added error catching to the initialization of the MobclixBrowserActivity to prevent badly constructed ads from crashing applications.

Version 3.1.0

  • Added Open Allocation support for the Millennial Media Android SDK. A network adapter is provided to make integration as simple as possible.
  • Added an addition check to prevent the Mobclix Android SDK from leaking Views and Activities.
  • Fixed a bug where fullscreen ads would link to the browser version of the Android Market, instead of redirecting to the Android Market app.
  • Fixed a bug where html ads that contained dropdown ads could cause a force close.

Version 3.0.1

  • Removed the ability to hard code a MobclixAdView at 320x50px, which would prevent proper scaling of a MobclixAdView on higher pixel density devices.
  • Fixed a bug where full screen AdViews would sometimes through NullPointerExceptions when clicked.

  • Added ProGuard support. For details on implementation, please refer to the Detailed Start Guide for Android.pdf file included with the Mobclix Android SDK.
  • Fixed a bug where Open Allocation Google AdMob would Force Close applications with an IllegalStateException.
  • Fixed an edge case bug caused by malformed ads attempting to load unsupported URIs.
  • Fixed an edge case bug that threw a NullPointerException when loading the MobclixBrowserActivity.
  • Reduced the memory footprint of the Mobclix Android SDK.

  • Fixed a bug where Full Screen creatives wouldn't function properly on tablets running Honeycomb.

  • Fixed a bug where ads wouldn't show up if Mobclix.onCreate(this); wasn't called to initiate Mobclix prior to the ad call.

  • MobclixFeedback now introduces a listener object to receive callbacks as to whether feedback was successfully sent.

  • Bug Fix: Fixed onLocationChanged NullPointerException.
    Mobclix AdViews would crash on a onLocationChanged call, caused by using location targeted ads.

Version 2.1.1

  • General: Improved thread management.
    Improved location thread management to reduce battery drain.
  • General: ACCESS_NETWORK_STATE permission has been made optional.
    ACCESS_NETWORK_STATE has been made optional to address concerns about permission requirements of the Mobclix Android Library.
  • General: Updated onOpenAllocation documentation.
    The documentation on onOpenAllocation of the MobclixAdViewListener was incorrect. It has been updated to reflect it's proper return value.
  • Bug Fix: Fixed Android Layout Editor bug.

  • Bug Fix: Demographics gps fix.
    Demographics data was failing to log when the location string returned by the phone was too long.
  • Bug Fix: Location provider fix.
    The Mobclix Android Library would occassionally trigger an IllegalArgumentException if the location provider was unavailable on the device. This was caused by a defect in Android as detailed at http://code.google.com/p/android/issues/detail?id=2163. The fix detailed has been applied.

Version 2.1.0

  • Feature: Added Analytics.
    The Mobclix Android Library now has better analytics support, better session tracking and custom event logging.
  • Feature: Added Feedback.
    The Mobclix Android Library can now be used to collect feedback information in the form of quantitative ratings, qualitative comments, and demographics information. Examples of this can be found in the Mobclix Android Demo application.
  • Feature: Better AdMob Open Allocation integration.
    The Mobclix Android Library no longer requires the developer to manually handle AdMob Open Allocation. By simply adding the AdMob SDK to the project in the Java Build Paths window, the Mobclix Android Library will automatically handle loading ads from AdMob. If no AdMob ad is available, the Mobclix Android Library will automatically request another ad.
  • General: The 300x50 AdView size has been changed to 320x50.
    The new AdView size allows the ads to be flush with the sides of most Android devices in portrait mode.
  • General: Rich Media ad improvements.
    Added support for two icon text ads. Added support for clicked ads to load in a minimal browser with a close icon.
  • General: Location handling improvements.
    Improved location targeting code to reduce battery drain and increase accuracy.
  • Bug Fix: Fixed OutOfMemoryException crash.
    Certain ads were causing the Mobclix Android Library to crash if the creatives being loaded were too large in size.

Version 2.0.4

  • Bug Fix: Fixed Android SDK 7 (2.1) ViewFlipper bug in Android SDK 8.
    Android SDK 7 introduced a bug where a ViewFlipper would cause errors if the orientation of the phone changes back and forth rapidly. The fix implemented is detailed at http://code.google.com/p/android/issues/detail?id=6191. The fix was extended to the recently released SDK version 8.
  • Bug Fix: Fixed bug with Android 1.5.
    The ViewFlipper fix used an API 4 method, causing AdViews to run into issues on Android platforms 1.5 and earlier.

Version 2.0.3

  • Feature: Added pause() and resume() methods.
    Some developers hoping to pause ad requests while they ran heavy processing threads were using cancelAd() to stop ads. This is inelegant, so pause() and resume() were introduced to allow for cleaner pausing and resuming of ad requests.
  • General: Improved location awareness of FINE and COARSE location permissions.
    Improved the handling of location awareness. Notably, the SDK will only attempt to require a GPS position for ten seconds before succeeding or giving up. This will prevent the SDK from draining the battery unduly.
  • Bug Fix: Fixed Android SDK 7 (2.1) ViewFlipper bug.
    Android SDK 7 introduced a bug where a ViewFlipper would cause errors if the orientation of the phone changes back and forth rapidly. The fix implemented is detailed at http://code.google.com/p/android/issues/detail?id=6191.
  • Bug Fix: Fixed does not refresh setting bug.
    Setting the refresh time on the Mobclix Developer Dashboard would prevent the AdView from loading on instantiation. However, if the activity lost and regained focus, the AdView would begin refreshing.

Version 2.0.2

  • General: Increased performance on remote configuration retrieval.
    Update to the polling thread that waits for remote configuration retrieval has been updated to be more efficient. Also improved remote configuration error handling when the device has no limited or no connection.
  • General: Improved text ad formatting.
    Single line text ads are now vertically centered in the ad space. Originally, they appeared above or below another TextView that only contained an empty string.
  • Bug Fix: Fixed Mobclix Suballocation.
    Suballocation requests were returning as -503 UNAVAILABLE errors instead of Suballocation for AdMob and Google AdSense.
  • Bug Fix: Better error handling when an ad is clicked.
    If an MobclixAdView was instantiated with a non-Activity context, an exception would be thrown when the ad was clicked. This has been handled and should now direct the user to the url.

Version 2.0.1

  • Bug Fix: Fixed a NullPointerException.
    Devices trying to load an add before the locale of the device was loaded were running into a NullPointerException.

Version 2.0.0

  • Feature: Rich Media ad support.
    The Mobclix Android Library now has support for Mobclix's Rich Media ad formats, including text ads, banner ads, slider ads, video ads, and popover ads.
  • Feature: Remote Configuration support.
    The Mobclix Android Library now has support for remote configuration of its ad units. From the dashboard, you can now set the refresh time for your Ad Units, which will be reflected every time the app relaunches.
  • General: Minimum refresh time set to 30 seconds.
    Lower refresh rates decrease CTR and can strain the system.
  • General: AdViews retrieve ads automatically upon initialization.
    AdViews no longer need to have getAd() called to retrieve ads. This means that no code at all is necessary to grab ads, only adding the AdView to the layout.
  • General: Method public boolean shouldTouchThrough(MobclixAdView adview) removed.
    This method allowing the blocking of ad touch through has been removed.
  • General: Reduced delay on loading first Activity with a Mobclix AdView by 80%.
    Better handling of retrieving phone settings has reduced the delay noticed by some developers when they first loaded an Activity with a Mobclix AdView.
  • Bug Fix: NullPointerException no longer thrown on query and keywords methods of MobclixAdViewListener.
    When the query or keywords methods of the MobclixAdViewListener were set to return null, the Mobclix library would throw a NullPointerException.
  • Bug Fix: NullPointerException no longer thrown on initialization of a Mobclix AdView when the APPLICATION_ID isn't set in the AndroidManifest.
    When the APPLICATION_ID wasn't set in the AndroidManifest, instantiating a Mobclix AdView was throwing a NullPointerException. This has been replaced with a NotFoundException with an error message to better indicate the problem.
  • Bug Fix: MobclixAdViewListener methods are now called in the UI Thread.
    In previous versions of the library, the MobclixAdViewListener methods were called in a separate thread, preventing developers from using these methods to change the UI of their application with implementing their own handlers. This has been fixed to make it easier to use these methods, especially for handling Open Allocation.
