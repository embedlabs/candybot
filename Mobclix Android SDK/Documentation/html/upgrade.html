New to Mobclix Android SDK 2.3

Full Screen Ad View

In Mobclix SDK 2.3, we have introduced the ability to display a fullscreen interstitial ad. It will allow you to preload and display a fullscreen advertisement in your application. Please see the quickstart guide and full SDK documentation to learn how to implement and use this new feature.

Rich Media and User Interaction

In Mobclix SDK 2.0 we introduced the ability to support autoplay interstitial ads, and we’ve taken it a step further in 2.2. Included as apart of our additional rich media support is the ability for rich media ads to automatically access features of the phone that wouldn’t otherwise take them away from your app. Some of these features include the vibrate functionality, audio support, and access to the LED light on supported Android devices. We’ve added a new configurable setting called richMediaRequiresUserInteraction. By default, this is turned off. You can turn it on by either going to the developer dashboard or returning "true" for the new method setRichMediaRequiresUserInteraction of MobclixAdViews. As with autoplay interstitial ads, you’ll see a much higher eCPM if you allow these kind of ads in your application.

Upgrading From Mobclix for Android 4.0.0

Recommended Changes:

Review the Detailed Start Guide for Android.pdf to review changes to how the Mobclix Android SDK handles Open Allocation. For those using Open Allocation: Other, review the section on the newly added 'continueRequest()' method.

Upgrading From Mobclix for Android 3.1.2

Recommended Changes:

Add the android:hardwareAccelerated=”true” line to your AndroidManifest.xml as shown in the Detailed Start Guide for Android.pdf.

Upgrading From Mobclix for Android 2.3.2

Recommended Changes:

Remove the GET_TASKS permission. It is no longer a required permission.

Check the ProGuard section in the Detailed Start Guide for Android.

See Recommended Changes for upgrading from 3.1.2.

Upgrading From Mobclix for Android 2.3.1

Recommended Changes:

Use the new Google AdMob SDK as found on their website. Please follow their instructions regarding changes to the AndroidManifest.xml. Additional information can be found in section 3.1 (Adding additional SDK's to your Application) of the Detailed Start Guide for Android.pdf.

See Recommended Changes for upgrading from 2.3.2.

Upgrading From Mobclix for Android 2.1.3

Required Changes:

Change the line:
<activity android:name="com.mobclix.android.sdk.MobclixBrowserActivity"/>

To:
<activity android:name="com.mobclix.android.sdk.MobclixBrowserActivity"
android:theme="@android:style/Theme.Translucent.NoTitleBar"/>


In the AndroidManifest.xml. An example of this can be found on the quickstart page.

Upgrading From Mobclix for Android 2.0.4

Required Changes:

Add the lines:
<uses-permission android:name="android.permission.READ_PHONE_STATE"></uses-permission>
To the AndroidManifest.xml directly after the other required permissions metadata. An example of this can be found on the quickstart page.

Add two new methods to any MobclixAdViewListeners:
public boolean onOpenAllocationLoad(MobclixAdView adView, int openAllocationCode) {
return false;
}

public void onAdClick(MobclixAdView adView) { return; }

See Required Changes for upgrading from 2.1.3.

Recommended Changes:

MobclixMMABannerXLAdView has been changed from a 300x50 ad unit to a 320x50 ad unit. This should make it flush to the sides of the screen while most Android devices are in portrait mode. Please adjust your layouts accordingly.
Analytics for the Mobclix Android Library has been turned on. It is recommended to make a call to Mobclix.onCreate(this); at the entry point of your application. Once called, the Mobclix Android library will track your application's sessions. For more accurate results, it is recommended to make a Mobclix.onCreate(this) call at every Activity's onCreate method and to make a Mobclix.onStop(this) call at every Activity's onStop method.

See Recommended Changes for upgrading from 2.3.1.

Upgrading From Mobclix for Android 1.0.3 - 1.0.4

Required Changes:

Add the line:

<activity android:name="com.mobclix.android.sdk.MobclixBrowserActivity"/>

To the AndroidManifest.xml directly after the APPLICATION_ID metadata. An example of this can be found on the quickstart page.
See Required Changes for upgrading from 2.0.4.

Recommended Changes:

getAd() calls are no longer required and it is recommended to remove it from your code.
setRefreshTime() calls are no longer required and it is recommended to remove it from your code. Please set your refresh time remotely on the Mobclix Dashboard.
public boolean shouldTouchThrough(MobclixAdView adview) has been deprecated and it is recommended to remove it from your MobclixAdViewListener.
Please note that unless you're setting up a MobclixAdViewListener, it is no longer necessary to get a reference to the AdViews in your Activity class at all.

See Recommended Changes for upgrading from 2.0.4.

Upgrading From Mobclix for Android 1.0.2

Required Changes:

Add methods to MobclixAdViewListeners: The ability to add keywords and a query to ad requests has been added to the MobclixAdViewListener. Please update any MobclixAdViewListeners with the keywords() and query() methods.
See Required Changes for upgrading from 1.0.3-1.0.4 and upgrading from 2.0.4.

Recommended Changes:

See Recommended Changes for upgrading from 1.0.3-1.0.4 and upgrading from 2.0.4.

Upgrading From Mobclix for Android 1.0.0

Required Changes:

Removed MobclixActivity.class: MobclixActivity.class has been removed to simplify integration. Just delete all references to com.mobclix.android.sdk.MobclixActivity.
See Required Changes for upgrading from 1.0.3-1.0.4 and upgrading from 2.0.4.

Recommended Changes:

See Recommended Changes for upgrading from 1.0.3-1.0.4 and upgrading from 2.0.4.
